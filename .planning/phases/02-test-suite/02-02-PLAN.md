**Test 1: "mission scenario produces complete flight lifecycle"**

1. Start broker, start collector on topics:
   - `sys/product/+/status`
   - `thing/product/+/state`
   - `thing/product/+/osd`
   - `thing/product/+/events`
2. Spawn simulator with `--scenario mission`.
3. Collect until the offline status message arrives (second message on status topic with sub_devices.length === 0), or timeout at 120000ms (mission scenario takes ~50s with 7 steps x 3 OSD cycles x 2s each).
4. Filter messages by topic type:
   - `statusMsgs` = messages on `sys/product/+/status`
   - `progressMsgs` = messages on `thing/product/+/events` where payload.method === 'flighttask_progress'
   - `osdMsgs` = messages on `thing/product/+/osd` (device OSD, not dock)
5. Assert:
   - statusMsgs.length === 2 (online + offline)
   - progressMsgs.length === 7 (one per MISSION_STEPS entry)
   - progressMsgs statuses in order: [2, 3, 5, 5, 5, 5, 6]
   - progressMsgs percents in order: [0, 5, 15, 35, 55, 80, 100]
   - Each progress message has data.output.ext.flight_id (string, starts with 'sim-flight-')
   - Each progress message has data.output.progress.current_step (number)
   - osdMsgs.length === 21 (7 steps x 3 OSD per step)
   - Final status message has sub_devices.length === 0

**Test 2: "HMS alarms correlate battery imbalance code to cell differential threshold"**

1. Start broker, start collector on topic `thing/product/+/events`.
2. Also subscribe to `sys/product/+/status` to detect scenario end.
3. Spawn simulator with `--scenario hms-alarm`.
4. Collect until offline status arrives. Timeout 30000ms (scenario takes ~16s: 500ms + 1000ms + 5 x 3000ms).
5. Filter for HMS events: payload.method === 'hms'.
6. Assert exactly 5 HMS events received.
7. For each HMS event, check payload.data.list for alarm code '0x16100086':
   - The hms-alarm scenario uses ticks [50, 100, 150, 200, 250].
   - cellDifferential(50) base = 60mV -- well below 200mV threshold, battery alarm absent.
   - cellDifferential(100) base = 120mV -- below threshold, absent.
   - cellDifferential(150) base = 180mV -- below threshold, absent (noise could push it over, so check conservatively).
   - cellDifferential(200) base = 240mV -- above threshold, battery alarm MUST be present.
   - cellDifferential(250) base = 300mV -- above threshold, battery alarm MUST be present.
   - Assert: the LAST 2 HMS events (indices 3 and 4) contain an alarm with code '0x16100086' and level 2.
   - Assert: the FIRST 2 HMS events (indices 0 and 1) do NOT contain code '0x16100086'.
   - (Index 2 / tick 150 is borderline due to noise -- do not assert on it.)
8. Assert: All 5 HMS events contain the base alarm code '0x16100001' (always present).
